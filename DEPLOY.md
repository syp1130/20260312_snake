# 배포 링크 공개 방법

## 비밀번호를 Vercel 환경 변수로 쓰기 (권장)

1. **Vercel 대시보드** → 프로젝트 선택 → **Settings** → **Environment Variables**
2. **Name:** `GAME_PASSWORD`  
   **Value:** `3082` (또는 원하는 비밀번호)  
   **Environment:** Production (필요하면 Preview도 추가)
3. **Save** 후 **Redeploy** (Deployments에서 맨 위 배포 ⋮ → Redeploy)

이렇게 하면 **Git 저장소에는 비밀번호가 없고**, Vercel 대시보드에서만 바꿀 수 있습니다.  
비밀번호 변경 시 환경 변수만 수정하고 재배포하면 됩니다.

> 로컬에서 `npm run build`를 실행하면 `public/index.html`이 수정될 수 있으니, 배포는 Vercel 빌드에만 맡기고 로컬에서는 빌드하지 않는 것을 권장합니다.

### 비밀번호 화면이 안 뜨는 경우

**한 번이라도 비밀번호를 입력해 입장한 적이 있으면** 브라우저가 “이미 통과”로 기억해서, 같은 탭에서는 비밀번호 창을 다시 안 띄웁니다.

- **해결 1:** **시크릿(프라이빗) 창**에서 배포 링크를 새로 열어 보세요.
- **해결 2:** 게임 화면이 보이면 **시작하기** 버튼 아래 **「비밀번호 다시 입력」** 링크를 누르면, 다음 로드부터 다시 비밀번호 창이 뜹니다.

---

## 1. Vercel 링크는 이미 공개됨

Vercel에 배포하면 **기본적으로 링크는 공개**입니다.  
아무나 URL만 알면 접속할 수 있습니다.

- 배포된 주소 예: `https://20260312-snake-git-master-xxxx.vercel.app`
- 이 링크를 팀원·지인에게 그대로 공유하면 됩니다.
- 접속한 사람은 **비밀번호 3082**를 입력한 뒤 게임을 할 수 있습니다.

즉, **별도 설정 없이** 링크를 공유하는 것만으로 “누구나 링크로 들어올 수 있는 공개” 상태입니다.

---

## 2. Vercel에서 더 열어두고 싶을 때

혹시 예전에 “배포 보호”를 켜 둔 경우에만 아래를 확인하면 됩니다.

1. [Vercel 대시보드](https://vercel.com) → 해당 프로젝트 선택
2. **Settings** → **Deployment Protection** (또는 **Security**)
3. **Vercel Authentication**, **Password Protection** 등이 켜져 있으면 **끄기**
4. 그러면 배포 URL은 비밀번호/로그인 없이 그대로 접속 가능 (우리 게임은 게임 내 비밀번호 3082만 있음)

---

## 3. 요약

| 하고 싶은 것 | 방법 |
|-------------|------|
| **링크 공개** (누구나 링크로 접속 가능) | Vercel 배포 URL 그대로 공유. 기본이 공개라 추가 설정 없음. |
| **팀원만 접속** | 링크 + 비밀번호 **3082**만 팀원에게 알려주기. |
| **비밀번호 없이 완전 공개** | 코드에서 비밀번호 화면을 제거해야 함 (원하면 요청 시 수정 가능). |

지금 상태: **링크는 이미 공개**이고, 들어온 사람은 **비밀번호 3082** 입력 후 게임 가능합니다.
